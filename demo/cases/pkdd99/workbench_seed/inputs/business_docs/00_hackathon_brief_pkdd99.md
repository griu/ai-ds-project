# Hackathon brief — PKDD’99 Financial

## Sponsor ficticio
Equipo de Advanced Analytics para originación de préstamos

## Necesidad
Se quiere usar un dataset relacional clásico con historial de transacciones para demostrar cómo debe plantearse un proyecto de scoring temporal cuando la fecha de corte es crítica.

## Decisión a apoyar
Evaluar el riesgo asociado a préstamos en el momento de originación.

## Qué se espera del equipo
1. Definir unidad de predicción y target
2. Fijar la fecha de corte correcta
3. Estructurar las fuentes relacionales
4. Definir backlog de conectividad, EDA y features
5. Evitar leakage temporal desde el inicio

## Qué no se pide todavía
- un modelo final perfecto;
- una explotación productiva real;
- integración con sistemas bancarios reales;
- una validación regulatoria completa.
