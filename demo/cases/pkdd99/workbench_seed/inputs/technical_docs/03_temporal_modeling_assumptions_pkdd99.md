# Supuestos de modelización temporal — PKDD’99

## Unidad de decisión esperada
Préstamo en fecha de originación, con posibilidad de usar `account_id` como eje de agregación según el framing final.

## Pregunta a resolver
¿Se puede estimar el estado esperado del préstamo usando únicamente el histórico conocido hasta la fecha de concesión?

## Uso esperado del output
- priorización analítica;
- entrenamiento metodológico;
- benchmark de modelización temporal;
- demostración de buenas prácticas.

## Requisitos para las siguientes fases
- bloquear explícitamente el uso de datos futuros;
- documentar el timestamp objetivo;
- dejar trazabilidad de cualquier agregación temporal;
- preparar una validación coherente con la fecha de corte.
