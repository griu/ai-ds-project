# Jury notes y objetivos de demo — Home Credit

## Qué debe quedar claro al público
- La estructura de 2 repos funciona de forma simple
- Claude no arranca generando código arbitrariamente
- El proyecto se define primero en términos de negocio, decisión y datos
- Las tareas se encadenan con `next_task.md` y `task_result.md`
- El método es reusable para otros datasets

## Qué debería poder contestar el equipo durante la demo
- cuál es la unidad de predicción;
- cuál es el target;
- qué información puede generar leakage;
- qué tablas históricas exigen agregaciones;
- cómo se piensa la validación temporal;
- cómo se pasaría después a industrialización.

## Señales de una buena ejecución
- documentos base claros;
- backlog ordenado;
- lenguaje de negocio correcto;
- separación nítida entre control y ejecución.
