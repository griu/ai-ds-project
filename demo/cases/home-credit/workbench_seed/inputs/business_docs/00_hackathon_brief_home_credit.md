# Hackathon brief — Home Credit

## Sponsor ficticio
Dirección de Riesgo Retail

## Necesidad
Se quiere rediseñar el punto de partida de un proyecto de scoring de nuevas solicitudes de crédito al consumo. El sponsor no quiere una “demo de modelo”, sino una demostración de cómo un data scientist puede usar Claude en VS Code para definir el proyecto correctamente desde el inicio.

## Decisión a apoyar
Priorizar y evaluar nuevas solicitudes de préstamo para apoyar procesos de riesgo, revisión y segmentación.

## Qué se espera del equipo
1. Estructurar el problema de negocio
2. Traducirlo a un marco de decisión analítica
3. Ordenar las fuentes de datos
4. Identificar riesgos de leakage y mala observabilidad
5. Diseñar el backlog inicial de construcción del proyecto

## Qué no se pide todavía
- un modelo final en producción;
- una estrategia regulatoria completa;
- automatización total del scoring;
- integración corporativa real.
