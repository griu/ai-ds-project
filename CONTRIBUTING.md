## How to contribute

1. Fork the repository.
2. Create a branch from `develop`: `git checkout -b feature/my-feature`.
3. Make your changes and commit them with a clear message.
4. Push your branch and open a Pull Request against `develop`.
5. Wait for review before merging.

## Branching model

- `main` — stable, tagged releases only.
- `develop` — integration branch for ongoing work.
- `feature/*` — feature branches, branched off `develop`.

## Commit messages

Use clear, concise commit messages in English or Spanish. Reference issues when applicable: `fix #123`.
