# Cuándo usar esta skill
Úsala cuando exista un `task_result.md` y haya que decidir la siguiente tarea.

# Procedimiento
1. Lee `task_result.md`.
2. Identifica:
   - objetivo de la tarea anterior
   - entregables generados
   - bloqueos
   - tareas pendientes
   - estado de la conectividad de datos
3. Evalúa si la tarea:
   - está finalizada
   - está parcialmente finalizada
   - debe rehacerse
4. Si la conectividad aún no está resuelta y condiciona el trabajo posterior, prioriza esa definición como siguiente paso.
5. Redacta `next_task.md` con:
   - objetivo
   - contexto
   - inputs
   - outputs esperados
   - criterios de finalización
   - límites de la tarea

# Regla
No mezcles dos fases grandes en una sola tarea.
