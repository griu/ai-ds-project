# Inputs

Coloca aquí los materiales brutos del proyecto:

- `business_docs/`: PDFs, Word o notas de negocio
- `technical_docs/`: documentación técnica, esquemas, normas
- `data_samples/`: muestras CSV, parquet o extractos
- `dictionaries/`: diccionarios de variables, layouts, glosarios

La primera tarea típica del proyecto será normalizar estos inputs en documentación operativa dentro de `docs/`.
Además, si la conectividad aún no está definida, una de las primeras tareas debe fijar qué combinación de fuentes y librerías se usará.
